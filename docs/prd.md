# 제품 요구 사항 정의서 (PRD): 루틴 체커 (Routine Checker)

## 1. 프로젝트 개요

"루틴 체커"는 사용자가 매일 약 10개의 습관을 추적하고, 한 달 동안의 진행 상황을 한눈에 시각화하여 루틴을 꾸준히 유지할 수 있도록 돕는 웹 애플리케이션입니다.

## 2. 목표

- 일일 습관 체크를 위한 심플하고 직관적인 인터페이스 제공
- 한 달간의 루틴 유지 현황을 한눈에 파악할 수 있는 시각화 제공
- 시각적인 진행 추적을 통해 사용자의 습관 형성 동기 부여

## 3. 핵심 기능

### 3.1 월별 그리드 뷰 (Monthly Grid View)

- 한 달 전체를 나타내는 그리드 형태의 UI (행: 습관, 열: 날짜)
- 완료된 습관에 대한 시각적 표시 (Animate UI 컴포넌트 활용)
- 한 달 전체 성과를 한눈에 볼 수 있는 상태 표시기

### 3.2 일일 습관 추적

- 매일 10개의 습관에 대해 간편하게 체크인/토글 기능
- 현재 월 내에서 지난 날짜의 기록 확인 및 수정 가능

### 3.3 습관 관리

- 습관 추가, 수정, 삭제 기능 (집중도를 위해 약 10개로 제한)
- 습관 이름 및 시각화를 위한 커스텀 색상 설정

### 3.4 월간 진행 상황 시각화

- 주간/월간 달성률 표시
- 개별 습관별 연속 달성 기록(Streak) 카운터

## 4. 사용자 경험 (UX) 및 디자인

### 4.1 시각적 디자인

- **모던 & 프리미엄 에스테틱**: 깔끔하고 활기찬 반응형 디자인 (라이트 모드 전용).
- **미니멀리즘**: 불필요한 장식을 제거하고 핵심 데이터(습관 추적)에 집중한 UI.
- **애니메이션**: [Animate UI](https://animate-ui.com/docs/components)를 적극적으로 활용하여 습관 체크, 버튼 클릭, 화면 전환 시 생동감 넘치는 마이크로 애니메이션 적용.

### 4.2 레이아웃

- **대시보드**: 현재 월의 그리드 뷰에 집중한 메인 화면
- **반응형 디자인**: 데스크톱과 모바일 브라우저 모두에 최적화. 모바일에서도 그리드를 편하게 볼 수 있는 레이아웃.

## 5. 기술적 요구 사항

- **Frontend**: **React (Vite)** 기반의 효율적인 개발 환경
- **State Management**: **Zustand**를 사용한 가볍고 직관적인 상태 관리
- **Styling**: **shadcn/ui** 및 **Animate UI** 기반의 컴포넌트 및 스타일링
- **데이터 저장**: **LocalStorage**를 사용한 브라우저 내 데이터 유지
- **SEO**: 기본 메타 태그, 시맨틱 HTML 적용 및 빠른 로딩 속도 확보

## 6. 로드맵

- **1단계**: React(Vite) 프로젝트 설정 및 shadcn/ui/Animate UI 환경 구축
- **2단계**: Zustand 상태 관리 및 LocalStorage 연동 (습관 데이터 구조 설계)
- **3단계**: Animate UI를 활용한 핵심 월별 그리드 및 체크 기능 구현
- **4단계**: 습관 관리 기능 추가 및 전체적인 디자인 폴리싱
