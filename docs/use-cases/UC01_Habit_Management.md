# UC01: 습관 관리 (Habit Management)

## 1. 개요

사용자가 추적하고자 하는 습관을 시스템에 등록, 수정 또는 삭제하는 과정입니다.

## 2. 관련 요구 사항

- PRD 3.3 (습관 관리)
- TRD 3.1 (Habit 모델)

## 3. 액터 (Actor)

- 사용자 (User)

## 4. 기본 흐름 (Success Scenario)

### 4.1 습관 추가

1. 사용자가 '습관 추가' 버튼을 클릭합니다.
2. 시스템은 습관 이름과 색상을 초기화할 수 있는 입력 창(모달)을 표시합니다.
3. 사용자가 습관 이름을 입력하고 원하는 색상을 선택합니다.
4. 사용자가 '저장' 버튼을 클릭합니다.
5. 시스템은 새로운 습관을 생성하고 리스트에 추가합니다. (Animate UI 효과 적용)
6. 시스템은 업데이트된 습관 목록을 LocalStorage에 저장합니다.

### 4.2 습관 수정

1. 사용자가 특정 습관의 '편집' 아이콘을 클릭합니다.
2. 시스템은 기존 습관 정보가 입력된 수정 창을 표시합니다.
3. 사용자가 이름을 변경하거나 색상을 수정합니다.
4. 사용자가 '저장' 버튼을 클릭합니다.
5. 시스템은 해당 습관 정보를 업데이트하고 화면에 반영합니다.
6. LocalStorage에 변경 사항이 동기화됩니다.

### 4.3 습관 삭제

1. 사용자가 특정 습관의 '삭제' 버튼을 클릭합니다.
2. 시스템은 삭제 확인 메시지를 표시합니다.
3. 사용자가 삭제를 확정합니다.
4. 시스템은 해당 습관 및 관련된 모든 일일 기록(Records)에서 해당 ID를 제거합니다.
5. 시스템은 리스트에서 해당 습관을 제거합니다. (삭제 애니메이션 적용)
6. LocalStorage에 최종 상태가 저장됩니다.

## 5. 예외 흐름 (Exception Scenario)

- **습관 개수 제한**: 습관이 이미 10개인 상태에서 추가를 시도할 경우, 시스템은 "최대 10개까지 등록 가능합니다"라는 안내 메시지를 표시하고 추가를 차단합니다.
- **이름 공백**: 습관 이름을 입력하지 않고 저장하려 할 경우, 시스템은 입력 요청 강조 표시를 합니다.
