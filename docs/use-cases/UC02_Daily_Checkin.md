# UC02: 일일 습관 체크 (Daily Habit Tracking)

## 1. 개요

사용자가 특정 날짜의 습관 달성 여부를 기록(체크/해제)하는 과정입니다.

## 2. 관련 요구 사항

- PRD 3.2 (일일 습관 추적)
- TRD 3.2 (Daily Record 모델)

## 3. 액터 (Actor)

- 사용자 (User)

## 4. 기본 흐름 (Success Scenario)

1. 사용자가 월별 그리드에서 특정 '날짜'와 '습관'이 교차하는 셀(Cell)을 확인합니다.
2. 사용자가 해당 셀을 클릭합니다.
3. **상태 전환 (토글)**:
   - **미완료 상태인 경우**: 시스템은 해당 날짜의 기록(`DailyRecord`)에 해당 습관 ID를 추가하고, 셀을 '완료' 상태로 시각화합니다. (Elastic/Check 애니메이션 적용)
   - **완료 상태인 경우**: 시스템은 해당 날짜의 기록에서 습관 ID를 제거하고, 셀을 '미완료' 상태로 되돌립니다.
4. 시스템은 변경된 기록을 Zustand 스토어에 업데이트합니다.
5. 시스템은 LocalStorage에 실시간으로 브라우저 데이터를 저장합니다.
6. 화면 상단의 달성률 지표가 즉시 업데이트됩니다.

## 5. 대안 흐름 (Alternative Scenario)

- **과거 날짜 수정**: 사용자는 현재 월 내의 지난 날짜 셀을 클릭하여 과거 기록을 자유롭게 수정할 수 있습니다.

## 6. 특이 사항

- 체크 작업은 별도의 저장 버튼 없이 클릭 즉시 반영되는 반응형 인터페이스를 지향합니다.
